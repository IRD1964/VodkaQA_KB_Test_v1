# 00 — Контекст предприятия

## Назначение раздела
Раздел формирует системный контекст Knowledge Base VodkaQA-Auditor.
Здесь фиксируются базовые понятия, принципы, логика FSMS и рамки
интерпретации для всех последующих разделов.

## Роль раздела в KB
Раздел используется как:
- концептуальная основа;
- точка выравнивания терминологии;
- источник аудиторской логики.

## Уровни знаний
- L1 — источники (Holah, ISO, Codex, регламенты) — *только как reference*;
- L2 — нормализованные требования FSMS;
- L3 — системная интерпретация для предприятия.

## Нормативные ориентиры
- ISO 22000:2018 — общая логика FSMS;
- Codex CAC/RCP 1-1969;
- НД Республики Узбекистан — приоритет;
- ТР ЕАЭС 047/2018 — справочно.

## Что входит в раздел
- системная цель безопасности;
- архитектура FSMS;
- роль PRP и HACCP;
- типы опасностей;
- системные риски и ограничения.

## Что НЕ входит
- детальные PRP;
- HACCP-анализ процессов;
- лабораторные методики;
- операционные СОП.
